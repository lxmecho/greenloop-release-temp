# 网站设计文档

## 1. 项目概述

项目名称：`绿循校园`

项目定位：面向校园场景的电子废物规范处理平台，提供“捐赠流转、固定回收点投放、上门回收”三种处理方式，并配套审核、公示、通知和积分激励机制。

项目目标：

- 为可继续使用的电子产品建立校内捐赠流转渠道
- 为损坏或老旧电子产品提供规范的回收处理路径
- 降低用户处理电子废物的操作门槛
- 为管理员提供一套清晰、高效且可持续扩展的管理后台

## 2. 角色与权限

系统包含两类核心角色：

- 普通用户：注册、登录、提交物品、查看进度、申请领取、公示查看、积分兑换
- 管理员：审核物品、处理申领、处理上门回收、管理积分商品、维护回收点、发布公告

权限设计原则：

- 前台操作以用户本人数据为边界
- 管理后台仅管理员可访问
- 公示页面不暴露手机号等敏感信息

## 4. 业务流程设计

### 4.1 捐赠流程

1. 用户提交可继续使用的设备，并选择“捐赠”
2. 管理员审核通过后，物品进入公示大厅
3. 系统向提交用户发放 5 积分
4. 其他用户可发起申领申请
5. 管理员审核申领并安排交接
6. 管理员标记流程完成

### 4.2 固定回收点流程

1. 用户提交损坏设备，并选择“投放固定回收点”
2. 用户填写校区、园区、具体点位和预计投放时间
3. 管理员审核通过后，记录进入“待投放”
4. 系统向提交用户发放 5 积分
5. 管理员在实际投放完成后标记“完成”

### 4.3 上门回收流程

1. 用户提交损坏设备，并选择“上门回收”
2. 用户填写宿舍楼栋、楼层、房间号和可预约时间段
3. 管理员审核通过后，记录进入“待上门回收”
4. 管理员上门完成交接后标记“已上门回收”
5. 系统在完成时向提交用户发放 5 积分

## 5. 状态流转设计

物品主状态：

- `pending_review`：待审核
- `published`：公示中
- `dropoff_ready`：待投放
- `pickup_scheduled`：待上门回收
- `matched`：已匹配
- `completed`：已完成
- `rejected`：已驳回
- `cancelled`：已取消

不同处理方式的主流转：

- 捐赠：`pending_review -> published -> matched -> completed`
- 固定回收点：`pending_review -> dropoff_ready -> completed`
- 上门回收：`pending_review -> pickup_scheduled -> completed`

## 6. 数据结构设计

当前系统采用面向记录的 JSON / MySQL 双存储兼容方案。核心数据集如下：

- `users`：用户信息、角色、校区、积分
- `items`：物品信息、处理方式、流程状态、回收地址/点位
- `applications`：公示物品申领申请
- `messages`：站内通知
- `rewards`：积分商品
- `redemptions`：积分兑换申请
- `pickup_points`：固定回收点信息
- `announcements`：首页公告

### 6.1 `items` 关键字段

- 基础信息：`title`、`category`、`brand`、`condition`、`description`、`image`
- 流程控制：`disposal_type`、`status`、`admin_note`、`points_awarded`
- 捐赠字段：`target_group`、`donation_reason`
- 固定回收点字段：`pickup_campus`、`pickup_zone`、`pickup_subpoint`、`pickup_display`、`pickup_time`
- 上门回收字段：`door_pickup_campus`、`door_pickup_building`、`door_pickup_floor`、`door_pickup_room`、`door_pickup_slot`

## 9. 安全与稳定性设计

- 使用 `csrf_token` 防止跨站请求伪造
- 用户密码使用哈希存储
- 管理员后台使用隐藏入口路径
- 公示页屏蔽敏感联系方式
- 上传图片限制类型与大小
- 旧数据通过迁移函数自动补齐新增字段，保证升级兼容性

## 10. 积分机制设计

设计目标：

- 激励用户参与电子废物规范处理
- 避免无效提交刷积分

当前规则：

- 捐赠：审核通过后发放 5 积分
- 固定回收点：审核通过后发放 5 积分
- 上门回收：管理员确认完成后发放 5 积分
- 兑换申请提交时先暂扣积分
- 管理员驳回兑换时自动退回积分

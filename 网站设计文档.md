# 网站设计文档

## 1. 项目概述

项目名称：`绿循校园`

网站访问地址：`https://134139.xyz/`

![网站访问二维码](assets/images/site-qrcode-poster.png)

项目定位：面向校园场景的电子废物规范处理平台，提供“固定回收点投放、上门回收”两种处理方式，并配套审核、通知和积分激励机制。

项目目标：

- 为损坏或老旧电子产品提供规范的回收处理路径
- 降低用户处理电子废物的操作门槛
- 为管理员提供一套清晰、高效且可持续扩展的管理后台

## 2. 角色与权限

系统包含两类核心角色：

- 普通用户：注册、登录、提交物品、查看进度、查看通知、积分兑换
- 管理员：审核物品、处理上门回收、管理积分商品、维护回收点、发布公告

权限设计原则：

- 前台操作以用户本人数据为边界
- 管理后台仅管理员可访问
- 页面不暴露手机号等敏感信息

## 4. 业务流程设计

### 4.1 固定回收点流程

1. 用户提交设备，并选择“投放固定回收点”
2. 用户填写校区、园区、具体点位和预计投放时间
3. 管理员审核通过后，记录进入“待投放”
4. 用户确认已投递后，回收人员统一回收并在入仓核验后按参考价分段发放积分
5. 管理员在实际回收完成后标记“完成”

### 4.2 上门回收流程

1. 用户提交设备，并选择“上门回收”
2. 用户填写宿舍楼栋、楼层、房间号和可预约时间段
3. 管理员审核通过后，记录进入“待上门回收”
4. 管理员上门取走物品并在系统中确认
5. 用户同步确认“已取走”后，记录进入入仓核验阶段
6. 管理员完成入仓核验后按参考价分段发放积分

## 5. 状态流转设计

物品主状态：

- `pending_review`：待审核
- `dropoff_ready`：待投放
- `dropoff_delivered`：已投递待回收
- `pickup_scheduled`：待上门回收
- `picked_up`：已取走待入仓
- `completed`：已完成
- `rejected`：已驳回
- `cancelled`：已取消

不同处理方式的主流转：

- 固定回收点：`pending_review -> dropoff_ready -> dropoff_delivered -> completed`
- 上门回收：`pending_review -> pickup_scheduled -> picked_up -> completed`

## 6. 数据结构设计

当前系统采用面向记录的 JSON / MySQL 双存储兼容方案。核心数据集如下：

- `users`：用户信息、角色、校区、积分
- `items`：物品信息、处理方式、流程状态、回收地址/点位
- `messages`：站内通知
- `rewards`：积分商品
- `redemptions`：积分兑换申请
- `pickup_points`：固定回收点信息
- `announcements`：首页公告

### 6.1 `items` 关键字段

- 基础信息：`title`、`category`、`brand`、`condition`、`description`、`image`
- 流程控制：`disposal_type`、`status`、`admin_note`、`points_awarded`
- 固定回收点字段：`pickup_campus`、`pickup_zone`、`pickup_subpoint`、`pickup_display`、`pickup_time`
- 上门回收字段：`door_pickup_campus`、`door_pickup_building`、`door_pickup_floor`、`door_pickup_room`、`door_pickup_slot`

## 8. 技术实现设计

技术方案：

- 后端：PHP 单入口页面应用
- 前端：原生 HTML + CSS + JavaScript
- 存储：JSON 文件存储，兼容 MySQL 存储驱动
- 上传：本地 `uploads/` 目录

核心文件：

- `index.php`：页面渲染与表单处理主入口
- `includes/bootstrap.php`：公共函数、状态文案、回收选项
- `includes/storage.php`：数据读写与初始化
- `includes/ui.php`：公共页面结构与前端交互脚本
- `assets/css/style.css`：视觉样式

## 9. 安全与稳定性设计

- 使用 `csrf_token` 防止跨站请求伪造
- 用户密码使用哈希存储
- 管理员后台使用隐藏入口路径
- 页面屏蔽敏感联系方式
- 上传图片限制类型与大小
- 旧数据通过迁移函数自动补齐新增字段，保证升级兼容性

## 10. 积分机制设计

设计目标：

- 激励用户参与电子废物规范处理
- 避免无效提交刷积分

当前规则：

- 所有处理方式均在最终完成环节录入参考价后再发放积分
- 积分分段建议：0-19 元=3 分，20-49 元=6 分，50-99 元=10 分，100-199 元=16 分，200-399 元=24 分，400-799 元=36 分，800-1499 元=50 分，1500 元及以上=68 分
- 管理员在最终核验时确认参考价，系统自动换算积分并发放
- 兑换申请提交时先暂扣积分
- 管理员驳回兑换时自动退回积分

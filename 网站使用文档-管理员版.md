## 3. 后台主要模块

后台包含以下模块：

- 概览
- 物品审核
- 上门回收
- 积分兑换
- 回收点管理
- 公告管理

## 4. 概览页面使用

在“概览”页中，可以快速看到：

- 待审核物品数量
- 待上门回收订单数量
- 待处理兑换数量
- 待入仓核验数量
- 启用中的回收点数量

## 5. 物品审核

进入“物品审核”后，可以看到所有 `待审核` 物品。

操作步骤：

1. 查看物品名称、类别、状态、说明和详情页
2. 在备注框填写审核意见
3. 点击“通过”或“驳回”

审核结果说明：

- 固定回收点：通过后进入“待投放”，生成物品编号
- 上门回收：通过后进入“待上门回收”，生成物品编号

驳回建议：

- 写清原因，方便用户重新提交
- 优先驳回图片不清晰、说明不完整或类型选择错误的记录

## 6. 上门回收订单处理

进入“上门回收”页签后，可以看到：

- 待上门回收订单
- 待入仓核验记录
- 已完成记录

待处理订单中会显示：

- 物品信息
- 提交人昵称与脱敏手机号
- 宿舍信息
- 预约时段

处理步骤：

1. 打开详情确认物品和预约信息
2. 按预约时间完成上门回收
3. 在备注中填写交接情况或回收时间
4. 点击“标记已取走”
5. 等待用户同步确认后完成入仓核验

## 7. 积分兑换处理

进入“积分兑换”页后，可以看到待处理兑换申请。

管理员可以：

- 通过兑换并填写领取说明
- 驳回兑换并退回积分

处理规则：

- 通过：状态改为 `已发放`
- 驳回：状态改为 `已驳回`，并自动退回积分、回补库存

## 8. 回收点管理

进入“回收点管理”后，可新增和查看固定回收点。

新增时需要填写：

- 回收点名称
- 校区
- 位置
- 开放时间
- 说明

管理建议：

- 保持开放时间与前台展示一致
- 回收点位置变动时及时更新
- 优先维护宿舍区附近的高频点位

## 9. 公告管理

公告会同步显示到首页公告栏，适合发布：

- 回收安排
- 商品领取说明
- 活动通知
- 特殊时间段服务调整

操作要点：

- 标题简洁明确
- 内容尽量写清时间、地点、对象
- 不需要展示时可切换为隐藏

## 10. 积分规则管理认知

当前积分规则如下：

- 固定回收点：完成最终核验后发放积分
- 上门回收：完成最终核验后发放积分

管理员在处理流程时要特别注意：

- 不要把“审核通过”误当成“流程完成”
- 只有完成投递或取走确认并核验后，才应发放积分

## 11. 常见管理场景建议

### 11.1 审核顺序建议

- 先处理待审核物品
- 再处理待上门回收订单
- 再处理积分兑换

### 11.2 备注填写建议

- 对用户可执行的信息写具体
- 对内部记录可写清交接情况
- 避免只写“已处理”“通过”等低信息量内容

### 11.3 日常检查建议

1. 检查默认管理员密码是否已修改
2. 检查回收点和公告内容是否为最新信息
3. 定期检查待审核、待上门回收、待入仓核验和待处理兑换记录
4. 如当前环境使用 JSON 存储，更新前先备份 `data/` 目录

## 12. 风险与注意事项

- 请勿在后台备注中填写用户完整隐私信息
- 处理上门回收前应先核对宿舍信息与预约时段
- 若当前环境使用 JSON 存储，请注意定期备份 `data/` 目录
- 正式上线时建议改用 MySQL 并配置真实短信服务
